# 英语分析技能

当用户发送英文段落或句子时，自动进行以下分析。

## 输出格式模板

```
## 📖 完整翻译（双语对照）

英文原文
对应的中文翻译

## 📝 逐词解析

| 单词/短语 | 词性 | 中文 | 解释/搭配 |

## 🔍 语法讲解

[句子结构分析、时态、语态、重点语法说明]

## 💡 重点词汇例句

**word1**:
- 英文例句 → 中文翻译
- 另一个例句 → 中文翻译
```

## 格式规范

### 1. 完整翻译（必须放在最开头）

- 先展示**英文原文**（可用引号包裹或原文呈现）
- 紧接**中文翻译**（完整句子，不是逐词堆砌）
- 长句可断句翻译，保持阅读流畅
- **双语对照**：左右或上下排列，便于对照学习

### 2. 逐词解析

| 单词/短语  | 词性     | 中文     | 解释/搭配      |
| ---------- | -------- | -------- | -------------- |
| 用表格展示 | 标注词性 | 中文意思 | 常用搭配或例句 |

- 重点词汇用**粗体**标注
- 常见搭配用`code`格式标注
- 同一词的不同含义列出 1-2 个常用义项

### 3. 语法讲解

- 分析句子主干结构（主语 + 谓语 + 宾语）
- 标注时态、语态
- 解释特殊句型或从句
- 用简单图示辅助说明（如箭头、层级）

### 4. 重点词汇例句

- 每个重点词汇给 **2-3 个实用例句**
- 例句要**情境化**（日常对话、编程、技术文档等）
- 例句后附中文翻译
- 标注不同场景的用法差异

## 示例

用户发送："Use comments sparingly. Only comment complex code."

输出：

```
## 📖 完整翻译（双语对照）

> Use comments sparingly.
> 注释要简洁克制。

> Only comment complex code.
> 只给复杂代码添加注释。

## 📝 逐词解析

| 单词/短语 | 词性 | 中文 | 解释/搭配 |
|-----------|------|------|------------|
| **use** | 动词 | 使用 | use sth. carefully 谨慎使用 |
| **comments** | 名词(复) | 注释 | code comments 代码注释 |
| **sparingly** | 副词 | 适量地 | use sparingly 少用 |
| **only** | 副词 | 只 | only + 动词 只做某事 |
| **comment** | 动词 | 加注释 | comment code 给代码加注释 |
| **complex** | 形容词 | 复杂的 | complex problem 复杂问题 |

## 🔍 语法讲解

**句型：祈使句 (Imperative)**

省略主语 "you"，表示建议和规范。

结构拆解：
┌─────────┐   ┌──────────┐   ┌────────────┐
│  Use    │ → │ comments │ → │ sparingly  │
│ 动词    │   │ 宾语     │   │ 方式状语   │
└─────────┘   └──────────┘   └────────────┘

**复数泛指**：`comments` 用复数表示泛指"所有注释"这类事物，
区别于单数指"一条具体注释"。

**分号连接**：两个相关独立句用分号连接，比句号更紧凑。

## 💡 重点词汇例句

**sparingly**:
- Use medicine sparingly. → 适量用药。
- Add salt sparingly when cooking. → 烹饪时少放盐。

**complex**:
- This is a complex problem to solve. → 这是一个复杂的难题。
- Don't make it more complex than it needs to be. → 不要把它弄得太复杂。
```

## 注意事项

- 翻译必须**通顺完整**，不是逐词硬翻
- 例句贴近**实际场景**（技术/日常/职场）
- 排版**清晰分区**，方便阅读
- 用户追问时，针对性解答
